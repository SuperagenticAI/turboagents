# Benchmarks

`turboagents` currently ships lightweight synthetic benchmark surfaces that are
safe to run on a development laptop. They are intended to validate the package
surface, compare bit-width trends, and catch regressions before heavier
benchmarking on larger hardware.

## Commands

```bash
turboagents bench kv
turboagents bench kv --format json
turboagents bench rag --format markdown
turboagents bench paper
```

## Current Built-in Datasets

- `tiny-kv`: small deterministic vector batch for KV-style reconstruction tests
- `tiny-rag`: small deterministic retrieval set for TurboRAG sanity checks
- `medium-rag`: reserved for a larger synthetic retrieval pass
- `paper-sim`: synthetic paper-style MSE comparison set

These are generated by `turboagents/bench/datasets.py`.

## What The Metrics Mean

- `mean_payload_bytes`: average serialized payload size for the compressed vector
- `mse`: mean squared reconstruction error after quantize/dequantize
- `mean_cosine_similarity`: cosine agreement between original and reconstructed vectors
- `ip_mae`: mean absolute error for inner products against a held-out query
- `recall_at_1`, `recall_at_10`: retrieval agreement against exact dot-product ranking

## What These Benchmarks Are Not

- They are not LongBench, Needle, BEIR, or MTEB.
- They are not end-to-end serving numbers for real models.
- They are not the final paper reproduction harness.

Those larger evaluations should run on the higher-memory Mac or another machine
with the required runtimes and datasets installed.

## Higher-Memory Mac Workflow

The repository now includes a reproducible benchmark harness for the larger Mac:

```bash
uv sync --extra rag --extra mlx
uv run python scripts/run_benchmark_matrix.py --output-dir benchmark-results/$(date +%Y%m%d-%H%M%S)
```

Optional real MLX run:

```bash
uv run python scripts/run_benchmark_matrix.py \
  --mlx-model mlx-community/Llama-3.2-3B-Instruct-4bit \
  --output-dir benchmark-results/$(date +%Y%m%d-%H%M%S)
```

Optional live pgvector validation:

```bash
uv run python scripts/run_benchmark_matrix.py \
  --pgvector-dsn postgresql://localhost/turboagents \
  --output-dir benchmark-results/$(date +%Y%m%d-%H%M%S)
```

Supporting scripts:

- `scripts/run_benchmark_matrix.py`
- `scripts/benchmark_mlx.py`
- `scripts/benchmark_rag_adapters.py`
- `scripts/benchmark_needle.py`
- `scripts/summarize_benchmark_results.py`
- `benchmarks/README.md`

## Minimal Long-Context Eval

The repository now includes a minimal Needle-style long-context harness for MLX:

```bash
uv run python scripts/benchmark_needle.py \
  --model mlx-community/Llama-3.2-3B-Instruct-4bit \
  --context-tokens 2048 4096 8192 \
  --output benchmark-results/needle-$(date +%Y%m%d-%H%M%S).json
```

Current scope:

- deterministic exact-match retrieval of a single secret string
- configurable context lengths
- configurable insertion positions
- bit-width sweep across the MLX path

Still out of scope:

- LongBench
- full Needle-in-a-Haystack reproductions
- multi-document or multi-hop tasks
- serving latency matrix integration

Latest artifact:

- `benchmark-results/needle-20260326-20260326-234159.json`

Observed behavior on `mlx-community/Llama-3.2-3B-Instruct-4bit`:

- exact-match retrieval succeeded at insertion fraction `0.1` across `2048`, `8192`, and `16384` token prompts for all tested bit-widths
- exact-match retrieval failed consistently at insertion fractions `0.5` and `0.9`
- `contains_needle` stayed positive for some `2048` and `4096` cases at insertion fraction `0.5`, but this is weaker than the exact-match criterion
- the current minimal harness therefore shows early-position recall, not robust long-context retrieval
- elapsed time scaled materially with context length, reaching about `15.6s` to `21.5s` per case at `16384` tokens
